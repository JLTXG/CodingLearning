# 一、题目描述

## **问题概括**

你拥有 `n` 个会议室，编号从 `0` 到 `n-1`。有一系列会议等待安排，每个会议 `i` 都有一个计划举办的时间段 `[start_i, end_i)`（包含开始时间 `start_i`，不包含结束时间 `end_i`）。所有会议的 `start_i` 互不相同。

会议分配必须严格按照以下规则进行：

1. **优先分配编号最小的空闲会议室**：在每个需要安排会议的时刻（无论是原定开始时间还是延期后的开始时间），总是将会议分配给当前未占用且编号最小的那个会议室。
2. **延期机制**：如果某个会议在其（原定或后续的）开始时间点，所有会议室都已被占用，则该会议将**延期**。其新的开始时间将推迟到最早有会议室空出来的时刻，并且会议的**持续时间（`end_i - start_i`）保持不变**。
3. **延期会议优先级**：当多个延期的会议在等待，且有会议室空闲时，优先分配给其中**原定开始时间（`start_i`）更早**的会议。

你的任务是模拟整个过程，并统计每个会议室最终举办的会议次数。需要返回**举办会议次数最多的会议室编号**；如果多个会议室举办次数相同且都是最多，则返回其中**编号最小**的那个。

## **示例核心说明**

- **示例1**：2个会议室，4个会议。通过模拟分配，会议室0和会议室1最终都举办了2次会议。根据规则，返回编号较小的 `0`。

    ```
    输入：n = 2, meetings = [[0,10],[1,5],[2,7],[3,4]]
    输出：0
    ```

- **示例2**：3个会议室，5个会议。模拟后，会议室1和会议室2都举办了2次会议，返回编号较小的 `1`。

    ```
    输入：n = 3, meetings = [[1,20],[2,10],[3,5],[4,9],[6,8]]
    输出：1
    ```

## **数据范围**

- 会议室数量 `n`: `1 <= n <= 100`
- 会议数量: `1 <= meetings.length <= 10^5`
- 单次会议时间: `0 <= start_i < end_i <= 5 * 10^5`
- 所有 `start_i` 的值互不相同。
