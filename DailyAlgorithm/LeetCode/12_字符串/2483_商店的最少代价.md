# 一、题目描述

## **问题概括**

你获得一个仅由字符 `'Y'` 和 `'N'` 组成的字符串 `customers`，其长度表示商店营业的总小时数。字符串中的每个字符对应一个小时的情况：

- 若 `customers[i] = 'Y'`，表示第 `i` 小时有顾客到达。
- 若 `customers[i] = 'N'`，表示第 `i` 小时没有顾客到达。

你需要选择一个关门时间 `j`（`j` 的取值范围是 `0` 到 `customers.length`，包含两端），使得商店在第 `j` 小时**及之后**停止营业。代价的计算规则如下：

1. 在开门期间（即前 `j` 小时），每出现一个没有顾客到达的小时（即 `'N'`），代价增加 `1`。
2. 在关门期间（即第 `j` 小时及之后的小时），每出现一个有顾客到达的小时（即 `'Y'`），代价增加 `1`。

你的目标是确定一个关门时间 `j`，使得总代价最小。如果存在多个代价最小的关门时间，则返回**最早**的那一个（即最小的 `j`）。

## **示例说明**

- **示例1**：`customers = "YYNY"`。长度为 4，计算每个可能的 `j`（0 到 4）的代价：
    - `j=0`（全程关门）：开门期间无小时，无代价；关门期间所有小时，有顾客的小时是索引 0,1,3，代价为 3。总代价 3。
    - `j=1`：开门期间（索引 0）有顾客（Y），无代价；关门期间（索引 1,2,3）有顾客的小时是 1 和 3，代价为 2。总代价 2。
    - `j=2`：开门期间（索引 0,1）都有顾客，无代价；关门期间（索引 2,3）有顾客的小时是 3，代价为 1。总代价 1。
    - `j=3`：开门期间（索引 0,1,2）中索引 2 为 N，代价 1；关门期间（索引 3）有顾客，代价 1。总代价 2。
    - `j=4`：开门期间（索引 0,1,2,3）中索引 2 为 N，代价 1；关门期间无小时，无代价。总代价 1。
        最小代价为 1，对应 `j=2` 和 `j=4`，取最早的 `j=2`。
- **示例2**：`customers = "NNNNN"`。最优关门时间是 0，代价为 0（全程关门，没有顾客到达，无代价）。
- **示例3**：`customers = "YYYY"`。最优关门时间是 4，代价为 0（全程开门，所有小时都有顾客，无代价）。

## **数据范围**

- 字符串长度在 `1` 到 `10^5` 之间。
- 字符串仅包含字符 `'Y'` 和 `'N'`。

## Java代码

```java
class Solution {
    public int bestClosingTime(String customers) {
        int flag = 0, lenStr = customers.length(), cus = 0;
        for(int i = 0; i < lenStr; i++){
            if(customers.charAt(i) == 'Y'){
                cus++;
            }
        }
        int minCus = cus;
        for(int i = 0; i < lenStr; i++){
            if(customers.charAt(i) == 'N'){
                cus++;
            } else {
                cus--;
            }
            if(minCus > cus){
                minCus = cus;
                flag = i + 1;
            }
        }
        
        return flag;
    }
}
```
