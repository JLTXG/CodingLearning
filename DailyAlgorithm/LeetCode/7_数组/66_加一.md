# 一、题目描述

给定一个表示 **大整数** 的整数数组 `digits`，其中 `digits[i]` 是整数的第 `i` 位数字。这些数字按从左到右，从最高位到最低位排列。这个大整数不包含任何前导 `0`。

将大整数加 1，并返回结果的数字数组。

 

**示例 1：**

```
输入：digits = [1,2,3]
输出：[1,2,4]
解释：输入数组表示数字 123。
加 1 后得到 123 + 1 = 124。
因此，结果应该是 [1,2,4]。
```

**示例 2：**

```
输入：digits = [4,3,2,1]
输出：[4,3,2,2]
解释：输入数组表示数字 4321。
加 1 后得到 4321 + 1 = 4322。
因此，结果应该是 [4,3,2,2]。
```

**示例 3：**

```
输入：digits = [9]
输出：[1,0]
解释：输入数组表示数字 9。
加 1 得到了 9 + 1 = 10。
因此，结果应该是 [1,0]。
```

 

**提示：**

- `1 <= digits.length <= 100`
- `0 <= digits[i] <= 9`
- `digits` 不包含任何前导 `0`。



# 二、解题思路/解题过程

今天是一道简单的题目，我之前也写过，不过之前是用C++写的（我又重新用C++写了一遍，“执行用时”提升了，“消耗内存”效果下降了！），其对应的“执行用时”不是很可观，但是“消耗内存”很可观；我今天用Java写完之后，“执行用时”还可以，“消耗内存”不怎么可观！（后面，我用Python写的代码提交之后，发现“执行用时”和“消耗内存”的效果都算是提交的记录里面最优的！）
思路如下，这道题目算是大整数题目，但是他简单在只在低位数上加一个“1”，所以可以考虑几个情况，如下：

- 常规情况：常规数字，例如：1234，这种就是无需考虑进位！
- 进位情况：需要考虑进位，不过也不用太麻烦，只需要找到离较低位9最近的高位即可，将该高位加1，然后将后面的低位置为0即可（不包括全是9的情况），然后在返回数组即可！
- 特殊情况：数组中的数字全是9，这个时候就需要开一个比原数组空间多1的整数数组，这个数组默认值全为0，只要将该数组下标为0的值改为1即可，然后返回该数组！
Java代码如下：

```java
class Solution {
    public int[] plusOne(int[] digits) {
        int digLen = digits.length;
        for(int i = digLen - 1; i >= 0; i--){
            if(digits[i] != 9){
                digits[i]++;
                for(int j = i + 1; j < digLen; j++){
                    digits[j] = 0;
                }
                return digits;
            }
        }
        int[] newDig = new int[digLen + 1];
        newDig[0] = 1;
        return newDig;
    }
}
```

## Python代码

```python
class Solution:
    def plusOne(self, digits: List[int]) -> List[int]:
        lendig = len(digits)
        for i in range(lendig - 1, -1, -1):
            digits[i] += 1
            digits[i] %= 10
            if digits[i] != 0 :
                return digits
        newdig = [0] * (lendig + 1)
        newdig[0] = 1
        return newdig
```
